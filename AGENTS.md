# AGENTS.md

Правила дальнейшей разработки проекта «Тренажер-самоучитель по картам Таро».

## Общие правила

- Интерфейс приложения должен быть на русском языке.
- Проект остается статическим мобильным HTML-приложением.
- Не добавлять backend, серверную логику, базы данных и авторизацию.
- Не добавлять npm, сборку, package.json и внешние библиотеки.
- Основной код держать в `index.html`.
- Не публиковать секреты, токены, приватные ключи или персональные данные.

## Разработка интерфейса

- Сначала проектировать мобильный опыт, затем адаптировать под большие экраны.
- Делать интерфейс простым, спокойным и удобным для регулярного обучения.
- Избегать перегруженных экранов и лишнего текста.
- Все элементы управления должны работать без сети после загрузки страницы.

## Код

- Использовать чистые HTML, CSS и JavaScript.
- Поддерживать понятную структуру внутри `index.html`: разметка, стили, затем скрипты.
- Комментарии добавлять только там, где они действительно помогают понимать логику.
- Не подключать внешние шрифты, CDN, изображения или API без отдельного решения.

## Git и деплой

- Основная ветка: `main`.
- Деплой: GitHub Pages из ветки `main`, папка `/root`.
- Перед коммитом проверять, что `index.html` открывается как обычный статический файл.

## CHANGELOG

- Перед каждым коммитом обновлять `CHANGELOG.md`: кратко описывать пользовательские изменения, затронутые страницы и важные технические детали.
